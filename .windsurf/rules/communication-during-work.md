# Communication During Work

- When making changes that were not explicitly discussed, **explain what you're about to change BEFORE making the change**. Brief is fine; surprise edits are not.
- If you're unsure about the **scope** of a change, ASK for clarification before implementing — do not assume.
- When a change reveals new questions or new scope, surface them to the user before silently expanding the work.
- After a cluster of edits or tool calls, give a short summary of what was actually done so the user can audit without re-reading every file.
